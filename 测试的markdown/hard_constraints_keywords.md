## Hard Constraints（本轮绝对不能违反）

1. 数据库 / 后端
   - 不要动数据库（不要新增/修改/删除任何表、字段、索引、约束）。
   - 不要改数据库结构（schema migration 只能在我明确要求时再做）。
   - 不要在后端新增写数据库的逻辑。
   - 不要修改后端 API 的路径、方法（GET/POST 等）或返回 JSON 结构。
   - 不要改后端业务逻辑：除非我明确说可以改，不要改 controller / service / repository 中已有的 if/else、校验、权限逻辑。

2. 代码范围
   - 这次只改前端，不改后端。
   - 只允许修改前端样式、交互和调用方式，不允许改 server 端代码。
   - 不要编辑 `backend/`、`server/`、`api/`、`db/` 目录下的任何文件。

3. 函数名 / 接口名
   - 必须保留函数名 `getTodoList`，不要改名。
   - 不要修改 `getTodoList` 的函数签名（参数列表、返回类型不变）。
   - 不要删除 `getTodoList`，也不要新增一个同名但不同含义的函数。
   - 不要修改 `todo_api.py` 里 `Todo` 模型的字段名（id/title/owner/due/completed 等）。

4. 外部副作用
   - 不要写入任何非项目内的文件路径（例如桌面、用户目录、系统盘）。
   - 不要调用任何外部网络 API（除非明确要求），例如第三方服务、随机 URL。
   - 不要修改日志文件的路径或全局日志格式配置。

5. 回答格式哨兵
   - 回答最后必须加上「喵」字结尾。
